# Contributing to Self-Healing Infrastructure Simulator

We welcome contributions from the community. To ensure a smooth collaboration, please follow these guidelines.

## Code Style
- Follow PEP 8.
- Use `colorama` for any new console output.
- Ensure all new features are covered by tests (or at least valid in the simulator loop).

## Pull Requests
1.  Fork the repo.
2.  Create a feature branch (`git checkout -b feature/amazing-feature`).
3.  Commit your changes.
4.  Push to the branch.
5.  Open a Pull Request.

## Philosophy
Keep it simple, keep it autonomous. We optimize for resilience, not raw performance.
